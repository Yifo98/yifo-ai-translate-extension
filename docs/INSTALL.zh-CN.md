# Yifo AI Translate 安装与使用

这是一款侧载版浏览器翻译扩展，适合自己和朋友在 Chrome/Chromium 浏览器里使用。扩展不会内置共享 API Key，每个使用者都需要填写自己的 GLM、DeepSeek 或 OpenAI-compatible 接口配置。

## 一、从源码构建

第一次使用需要先安装依赖并生成 `dist/`：

```bash
npm install
npm run build
```

构建完成后，`dist/` 就是浏览器要加载的扩展目录。

如果要生成可分享 zip 包：

```bash
npm run release
```

生成的压缩包在 `release/` 目录。把 zip 发给朋友后，让对方先解压，再加载解压出来的扩展目录。

## 二、导入 Chrome / Chromium

Windows 和 macOS 的 Chrome、Edge、Brave 等 Chromium 浏览器步骤基本一致：

1. 打开浏览器地址栏，输入 `chrome://extensions`。
2. 打开右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本项目构建后的 `dist/` 目录。
5. 建议把 `Yifo AI Translate` 固定到工具栏，方便随时打开弹窗。

如果浏览器提示访问网站权限，请允许扩展访问需要翻译的网页。API Key 只保存在本机扩展存储中，不会提交到仓库，也不会打包进源码。

## 三、首次配置

1. 点击工具栏里的 `Yifo AI Translate` 图标。
2. 在“接口”页选择 Provider：
   - `DeepSeek`
   - `GLM Coding Plan`
   - `OpenAI-compatible`
3. 填写自己的 `Base URL`、`API Key` 和模型。
4. 点击“连通测试”，确认接口可用。
5. 点击“刷新模型”，如果服务商支持 `/models`，扩展会自动拉取模型列表。
6. 回到“页面”页，点击“翻译此页”。

## 四、常用功能

- `翻译此页`：翻译当前页面。
- `恢复原文`：移除译文并恢复页面原文。
- `双语对照`：原文和译文同时显示。
- `仅看译文`：尽量用译文替换原文显示。
- `以后总是翻译`：记住当前站点，后续自动翻译。
- `以后不翻译`：记住当前站点，后续不自动翻译。
- `侧边一键按钮`：页面右侧常驻一个小按钮，点一下直接翻译；翻译中再点会停止并恢复原文。

## 五、常见问题

### 点击翻译没有反应

先打开扩展弹窗确认：

- 当前站点是否已授权。
- Provider 是否已选择。
- API Key 是否已填写。
- 模型是否存在。
- 是否被站点规则设置为“永不翻译”。

### DeepSeek 只显示少量模型

扩展优先使用服务商官方 `/models` 返回结果。DeepSeek 官方通常只公开 `deepseek-chat` 和 `deepseek-reasoner` 这类模型名；如果后台没有在 `/models` 返回新模型，扩展不会凭空显示。

### 翻译很慢

网页翻译会按段落分批请求模型。速度主要取决于网页段落数量、模型延迟、当前服务商限流和网络状态。页面越大、段落越多，等待越久。

### 分享给朋友安全吗

可以分享源码或 release zip，但不要分享自己的完整配置导出文件。默认安全导出不包含 API Key；完整导出会包含敏感信息，只适合自己备份。
